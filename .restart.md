- **v0.3.1** — `OllamaAdapter` memoizes model presence + `show()` info per
  process (was 2 `show()` round-trips per job on the hot path; now one on a
  model's first job, zero after). `force=` bypass; fresh CLI starts cold. Test
  added (no Ollama). Closes the "extra pull+show on hot path" deferred item.
- **v0.3.2** — Two of the long-deferred hardening gaps closed:
  1. **`cancel()` now interrupts a *running* job**, not just a queued one.
     In-flight job tasks tracked per `job_id` (`self._inflight` +
     `self._worker_loop_ref`); `cancel()` (main thread) interrupts the executing
     task on the worker loop via `loop.call_soon_threadsafe(task.cancel)`. The
     CANCELLED result is written first; `_process_job` has a race guard that
     refuses to overwrite a cancellation and treats `CancelledError` distinctly
     so an interrupted job never becomes FAILED.
  2. **SQLite store uses WAL + `busy_timeout=5000`** (write-ahead logging: a
     reader no longer blocks the worker's writes; contending writers wait briefly
     instead of "database is locked"). Matters once a submitter + worker (or two
     worker processes) share the file. Plain-language comment in store.py.
- **v0.3.3** — Grok read-only review of the 0.3.2 hardening found 3 cheap
  same-process fixes; shipped: `except Exception` path in `_process_job` now
  re-checks status (won't clobber CANCELLED→FAILED); poll loop skips a `job_id`
  already in `_inflight` (no double-spawn); corrected a shutdown-drain comment
  (cancelled stragglers are intentionally left PROCESSING and re-queued by
  startup recovery per DQ-004 — no terminal result written there).
- **v0.4.0 — MongoDB backend (ADR-017; fulfils the ADR-002 option / DQ-010
  `MongoJobStore` promise).** Drop-in `JobStore` selected with
  `Hoglah(config={"backend":"mongo", ...})` or `HOGLAH_BACKEND=mongo`
  (`mongo_uri`/`mongo_db`/`mongo_collection`; defaults
  `mongodb://localhost:27017` / `hoglah` / `jobs`). `pymongo` is an **optional**
  extra (`pip install "hoglah[mongo]"`), lazy-imported — SQLite stays the
  zero-dep default. Why a server backend (3 operator-stated motivations, all
  confirmed): (1) **no single-file lock** — the WAL/busy_timeout dance doesn't
  apply; `claim_for_processing` is atomic server-side via `find_one_and_update`
  (QUEUED→PROCESSING), so concurrent workers (even cross-machine) run each job
  once; (2) **external visibility** — jobs are native documents
  (request/result/tags are sub-documents, not opaque JSON blobs), inspectable
  from `mongosh`/Compass; (3) **multi-machine workers** share one queue.
  Returned rows mirror `SQLiteJobStore` exactly (parsed `request`/`result`/`tags`
  + the raw `*_json` strings the client reads), so it's a true drop-in. New
  files: `src/hoglah/mongo_store.py`, `tests/test_mongo_store.py` (gated contract
  + full client e2e behind `RUN_MONGO_TESTS=1`, throwaway collection — both PASS
  against the real local Mongo). Fixed a pre-existing unbalanced README code
  fence in passing. **39 tests pass (incl. 2 Mongo), ruff clean. v0.4.0
  verified live on PyPI** (`pip install "hoglah[mongo]==0.4.0"` from a clean venv
  installs + imports `MongoJobStore`).
- **v0.4.1 — MongoDB backend hardening from a grok read-only review of 0.4.0**
  (no Critical findings). Three code changes in `mongo_store.py`: `_doc_to_dict`
  re-serializes with `json.dumps(default=str)` for exact SQLite parity (removes a
  latent `TypeError` on get()/list() for hand-inserted/migrated docs); the atomic
  claim pins `return_document=ReturnDocument.BEFORE` explicitly (was the pymongo
  default — guards the safety-critical line against a future default flip); the
  standalone `status` index became a compound `(status, priority, created_at)`
  covering the worker's poll filter+sort in one (status prefix still serves
  status-only lookups), `(priority, created_at)` retained for unfiltered list().
  Gated suite grew **2→7**: list filters (status/tags `$all`/parent) + priority
  ordering + offset/limit, `update_status(error=None)` COALESCE preservation,
  `delete_jobs` by status and `before=`, embedding round-trip, and a **16-thread
  concurrent-claim race proving exactly one winner**. **44 tests pass (incl. 7
  Mongo against real local Mongo), ruff clean. v0.4.1 verified live on PyPI**
  (clean-venv `pip install "hoglah[mongo]==0.4.1"` installs + imports). Skipped
  (intentional, per review): error-clear-on-success is SQLite parity not a bug;
  per-instance MongoClient / connect-timeout are operational nice-to-haves.

Remaining deferred (pre-existing, non-blocking): on the **SQLite** backend the
duplicate-*worker* hazard on one shared DB still needs a process-level lock
(submitter case is already safe via ADR-016; Mongo's server-side atomic claim
sidesteps this entirely).

## 2026-06-13 — Mahalath integration: embedding jobs (chunk H1 of 3)

Sister-domain work: making Hoglah usable as a **separate worker daemon** that
Mahalath feeds off a shared queue (Mahalath submits → polls an output folder
or gets an HTTP callback). Three Hoglah chunks: **H1 embedding jobs (DONE)**,
H2 output-folder delivery, H3 outbound callback URL.

- **H1 (embedding jobs) — DONE.** ADR-013. New job `kind` (`generate`|`embed`);
  `kind="embed"` carries input text in `prompt` → worker routes to
  `adapter.embed()` → `JobResult.embedding` + `embedding_dim` (output None).
  `OllamaAdapter.embed` (client.embed `/api/embed`, rejects NaN/Inf),
  deterministic `StubAdapter.embed` (dim 8). `Hoglah.submit_embedding(text,
  model=...)`. Tests 30 (+4) + 1 gated; ruff clean. NOT yet released.
- **H2 (output-folder delivery) — DONE.** ADR-014. Optional `output_dir`
  config (env `HOGLAH_OUTPUT_DIR`); worker writes `<output_dir>/<job_id>.json`
  atomically (temp + `os.replace`) on terminal status via new `_deliver()`
  hook, wired at all terminal `set_result` sites (worker success/error +
  `cancel()`). `ensure_dirs()` creates it. Best-effort; `None`=disabled.
  Tests 32 (+2) + 1 gated; ruff clean. NOT yet released.
- **H3 (outbound callback) — DONE.** ADR-015 (supersedes the V1 "no
  webhooks / callback_url is V2" non-goal). Per-job `callback_url`
  (on `submit`/`submit_embedding`); worker POSTs terminal `JobResult` JSON
  inside the same `_deliver()` hook, on a **daemon thread** so a slow/dead
  endpoint + retry backoff never block the worker loop or `cancel()`. Config
  `callback_max_retries` (3) + `callback_timeout_seconds` (10); exp. backoff;
  4xx stops early; failure logs + leaves the output file as fallback. Stdlib
  urllib only. Tests 34 (+2: e2e http server + give-up) + 1 gated; ruff clean.
